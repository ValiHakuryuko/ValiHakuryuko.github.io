Getting Started with Burp Suite

July 23, 2025

This post covers the basics of Burp Suite Community Edition and an explanation of all tools found in it.

Proxy

The Proxy tab in Burp Suite lets you intercept, view, and modify HTTP(S) traffic between your browser and the web server. It’s essential for analyzing requests/responses during web application testing. Some features are:

  1. Intercept: Pause and modify live traffic.
  2. HTTP History: View a log of all requests/responses.

Repeater

The Repeater tab lets you manually modify and resend HTTP(S) requests to a server. It’s used for testing inputs, analyzing responses, and repeating requests without affecting live traffic.

Sounds kind of similar to Proxy, doesn't it..? Well the difference lies as follows:

  1. How to send requests: Ctrl+R key or left click and "Send to Repeater".
  2. Sending Requests: Requests can be sent by the user to the html page repeatedly via this tab, making changes to them to check the server response.
  3. Edit: Try changing headers, body, cookie, anything to see how a server hanldes the change.
  4. Compare: Compare responses side-by-side.

Intruder

The Intruder tab is used to automate sending many customized requests to a target, useful for tasks like Brute Forcing, Fuzzing, Enumeration, etc.

We have to provide:

  1. Target: Mostly Proxy or HTTP History. Attack Type.
  2. Payloads: Data to insert/test.
  3. Attack Type: Sniper, Battering ram, Pitchfork, Cluster bomb.
